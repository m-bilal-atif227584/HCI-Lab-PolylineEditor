# 1. System Overview

The system is a **Polyline Drawing and Editing Application** that allows users to:

* Create polylines using mouse input  
* Edit vertices (move, delete, insert)  
* Control operations through keyboard shortcuts  

From an HCI perspective, the system focuses on:

* Supporting user tasks efficiently  
* Providing real-time interaction  
* Ensuring continuous visual feedback

---

# 2. Interaction & Task Analysis

## Primary User Tasks
1. Create polyline  
2. Edit polyline  
3. Manage drawing  

## Task Breakdown

* **Create:** Press `B` and click to add vertices  
* **Move:** Select vertex → reposition  
* **Delete:** Remove nearest vertex  
* **Insert:** Add point on segment  
* **Refresh:** Redraw canvas  
* **Quit:** End session  

## Interaction Characteristics

* Mouse → pointing and dragging  
* Keyboard → command execution  

### HCI Insights:
* Direct manipulation improves **learnability**  
* Keyboard shortcuts improve **efficiency**  
* Visual feedback ensures better **usability**

---

# 3. User Analysis

## Target Users
* Students learning graphics  
* Beginner designers  
* General computer users  

## User Characteristics
* Basic computer literacy  
* Familiar with mouse interactions  
* Prefer intuitive interfaces  

## User Goals
* Quickly create shapes  
* Easily modify polylines  
* Avoid errors during interaction

---

# 4. Conceptual Model & User Expectations

The system aligns with user mental models:

* Click → Create  
* Drag → Move  
* Delete → Remove  

### Importance:
* Reduces learning effort  
* Improves usability  
* Minimizes user errors

---

# 5. Constraints & Challenges

## Technical Constraints
* Managing multiple polylines efficiently  
* Real-time rendering requirements  

## Interaction Constraints
* Difficulty in precise vertex selection  
* Mouse accuracy limitations  

## Usability Challenges
* Users may select wrong vertex  
* Keyboard commands may be confusing initially

---

# 6. Usability & Interaction Considerations

Based on HCI principles:

* System should provide **instant feedback**
* Controls must be **simple and consistent**
* Important elements should be **visually highlighted**
* System should support **error tolerance (undo, confirmation)**

---

# 7. Design Implications from Analysis

The analysis suggests:

* Clear visualization of vertices  
* Immediate system response  
* Consistent interaction patterns  
* Support for user error recovery  

The design process should be **iterative**, allowing improvements based on user feedback.

---

# 8. Lessons Learned

* Analysis is crucial before design and implementation
* User-centered thinking improves system usability
* Task analysis helps in simplifying complex systems
* Interaction design is not only about UI but also about user behavior

---

# 9. Confusions

* Whether keyboard-based commands are sufficient or GUI controls should be added
* Level of complexity required for additional features (e.g., undo, insert point)
* How detailed the analysis should be for academic vs real-world projects
